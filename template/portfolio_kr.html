Gdengine, 게임 기획 엔진$gitlink(simhyeon/gdengine)


Gdengine(지디엔진)은 게임 기획 문서 생성 도구 및 엔진임. 지디엔진은 전용 파일인 gddt를 컴파일하여 다양한 게임 기획 문서 포맷으로 렌더링함.

사용자는 하나의 gddt 파일만 작성하면 되며 지디엔진에서 지원하는 다양한 매크로를 통해서 원소스 멀티 유즈가 가능함. 모든 렌더 과정은 하나의 스크립트를 통해서 자동화할 수 있으며 html,pdf,png,pptx 등으로 변환할 수 있음


Rif, 영향 관리 시스템$gitlink(simhyeon/rif)

Rif(리프)는 영향 관리 시스템으로서 서로 복잡하게 연결되어 있는 문서들의 의존 관계를 저장하고 문서 변경에 따른 취약성을 사용자에게 고지하는 역할을 함.

게임 기획 문서는 서로 강력한 상호 의존성을 가지고 있는 반면 특정 문서의 변경이 가져오는 영향을 명확하게 추적하고 계산하기 쉽지 않음. 이것을 해결하기 위한 프로그램이 리프임. 리프는 모든 문서 변경 내용을 추적하고 변경된 문서에 의존성을 가지고 있는 문서들의 상태를 새로 업데이트함으로써 프로젝트의 예측 가능성을 높임.



게임 개발


리들러: 온라인 카드 게임

게임 설명 영상

리들러는 Heroes of Might and Magic VII라는 상용 게임 리소스를 사용하여 만든 팬 게임으로서 Unet을 통한 매칭 메이킹을 통해서 온라인 플레이가 가능한 멀티 플레이어 게임.

리들러의 핵심 메카닉은 텍사스 홀덤 포커 기반의 카드 플레이로서 다음의 논리 구조를 가지고 있음.

  • 플레이어는 손패와 커뮤니티 카드를 조합하여 덱 스킬을 사용함
  • 플레이어는 매 턴마다 베팅을 하여 자기 덱 스킬의 효과를 강화함.
  • 덱 스킬의 효과는 베팅한 자원에 따라 크게 달라지므로 플레이어는 커뮤니티 카드, 상대의 베팅 등을 통해서 상대가 사용하고 하는 스킬을 유추하게 됨

테트리스 게임$gitlink(simhyeon/tetrisgame)


유명한 고전 게임 테트리스를 Rust 프로그래밍 언어와 Amethyst라는 데이터 지향 프로그래밍 기반의 게임 엔진으로 개발한 게임

데이터 지향 엔진으로 개발되었기 때문에 모든 시스템이 독립적으로 작동함. 그 결과 시스템 버그의 원인을 추적하기 용이하고 게임 오브젝트의 예측 불가능성이 감소함.

모든 시스템이 독립적이기 때문에 자연스럽게 멀티 코어 활용이 자연스러움

게임 서버 개발$gitlink(simhyeon/cardserver)


데모 영상


서버의 구조는 한 클라이언트가 서버 역할을 하는 p2p가 아닌 모든 로직을 계산하는 독립적인 서버로서 구성됨(dedicated)

서버는 동일한 방 ID를 가지고 있는 2개의 클라이언트를 하나의 커넥션으로 연결하고 각각의 신호를 중개하여 게임을 진행시키는 stateful 구조를 가지고 있음.
